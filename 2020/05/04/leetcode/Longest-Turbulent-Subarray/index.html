Longest Turbulent Subarray

A subarray A[i], A[i+1], ..., A[j] of A is said to be turbulent if and only if:

  • For i <= k < j, A[k] > A[k+1] when k is odd, and A[k] < A[k+1] when k is even;
  • OR, for i <= k < j, A[k] > A[k+1] when k is even, and A[k] < A[k+1] when k is odd.

That is, the subarray is turbulent if the comparison sign flips between each adjacent pair of elements in the subarray.

Return the length of a maximum size turbulent subarray of A.

思路

根据定义，turbulent数组有两种，可以分别对这两种定义的子数组求解，然后取较长的作为结果返回即可。因为是跟后一个元素比较，我们可以反向遍历数据。当索引i是奇数时，考虑A[i]与A[i+1]的大小，如果A[i] > A[i+1]则满足条件1，不满足条件2；bianl反之，满足条件2不满足条件1。当索引i是偶数时，判定方式类似。我们用一个变量存储满足条件的第一个索引，反向遍历数组，如果发现条件满足，则对当前定义的子数组的长度进行一次校验（当前值与计算值取较大者）。

上述思路只是按照题意在求解，没有将题目进行抽象。参考solution发现，对A[i-1], A[i], A[i+1] 进行比较，假定结果为-1, 1, -1, 对于任意一种定义，实际上只要是这种-1和1交替出现的情况是满足定义的，不用管是1还是-1开头。我们可以对数列进行一次比较，假定得到的是[-1, 1, 1, -1, 0, 1, -1]，题目可以归结于求最长的[1, -1, 1, -1]这种模式的子数字的长度。

上述代码还能优化，因为我们不必先把所有的比较结果先算出来再来进行遍历，因为当前的计算只涉及到3个元素，可以通过滑动窗口进行优化。
